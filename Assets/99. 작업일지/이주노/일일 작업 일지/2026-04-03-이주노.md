## 1. 작업 개요

> Addressables를 활용한 턴 순서 UI(TurnUI)를 구현하고, 턴 큐(Turn Queue) 관리 로직을 고도화(동속도 정렬 및 사망 처리)했습니다.

- **주요 작업 영역 :** 턴 UI 시각화, Addressables 비동기 에셋 로드 적용, 턴 정렬 및 사망자 처리 로직 개선
- **작업 목표 :** 전투 턴 순서를 시각적으로 보여주는 UI 시스템 구축 및 안정적인 턴 큐 업데이트 환경 마련

## 2. 문제 사항 / 변경 사항 / 주요 성과

### 📌 이슈 #1 : 턴 UI 초상화 로드 시 이미지 겹침/깜빡임 현상 및 메모리 관리

#### 문제 사항
- **문제 설명 :** 턴 순서가 변경될 때마다 UI 슬롯에 새로운 초상화 이미지를 비동기로 불러와야 함. 로딩이 완료되기 전까지 이전 턴의 이미지가 남아있어 시각적 불쾌감을 주고, 여러 번 로드 시 메모리 누수가 발생할 위험이 있음.

#### Before / After

| 구분 | 내용 |
|------|------|
| **Before** | 턴 순서 변경 정보가 콘솔에만 출력됨 (UI 부재) |
| **After** | Addressables 비동기 로드 시, 로딩 중에는 Image를 끄고 로드 완료 시 켜는 방식으로 깜빡임 해결. 배열 기반 핸들 관리로 메모리 릴리즈 적용. |

```csharp
// After (TurnUI.cs 핵심 로직)
// 1. 이미지가 로드되는 동안 기존 이미지를 감춤
portraitSlots[i].enabled = false;
LoadPortraitAsync(i, portraitSlots[i], currentQueue[i].ImageAddress).Forget();

// 비동기 로드 메서드 내부
if (_imageHandles[index].IsValid()) {
    Addressables.Release(_imageHandles[index]); // 이전 핸들 메모리 해제
}
_imageHandles[index] = Addressables.LoadAssetAsync<Sprite>(address);
await _imageHandles[index].ToUniTask();

if (_imageHandles[index].Status == AsyncOperationStatus.Succeeded) {
    targetImage.sprite = _imageHandles[index].Result;
    targetImage.enabled = true; // 로드 완료 후 이미지 출력
}
```

#### 주요 성과 및 해결 이유 (Why)

- **선택 이유 :** 단일 핸들이 아닌 **슬롯 개수만큼의 독립적인 핸들 배열(`_imageHandles`)**을 구성하여 각 슬롯이 개별적으로 메모리를 관리하고 릴리즈할 수 있도록 설계했습니다.
- **최종 선택 근거 :** `gameObject.SetActive(false)` 대신 `Image.enabled = false`를 사용하여 레이아웃 그룹 내에서의 자리 차지는 유지하면서 이전 이미지가 노출되는 것을 방지, 부드러운 UI 갱신을 달성했습니다.

### 📌 이슈 #2 : 동속도 유닛의 턴 정렬 및 실시간 사망 처리 로직

#### 문제 사항
- **문제 설명 :** `Speed`가 완전히 동일한 유닛 간의 턴 정렬 순서가 보장되지 않았고, 턴 진행 중 유닛이 사망했을 때 큐에서 제거되지 않는 문제가 있었음.

#### Before / After

| 구분 | 내용 |
|------|------|
| **Before** | Speed만 비교하여 동속도 시 정렬 결과가 불안정, 사망자 턴 스킵 부재 |
| **After** | `RandomSpeed` 기반 보조 정렬 추가 및 라운드 내 실시간 턴 큐 갱신 로직 추가 |

```csharp
// After (TestPlayer.cs의 정렬 보정)
public int CompareTo(ITurnUseUnit other)
{
    if (other == null) return 1;
    int speedComparison = other.Speed.CompareTo(this.Speed);
    
    // 스피드가 같다면 난수로 배정받은 RandomSpeed로 순서 결정
    if (speedComparison == 0) return other.RandomSpeed.CompareTo(this.RandomSpeed);
    
    return speedComparison;
}

// After (TurnManager.cs의 큐 관리 로직)
private void RemoveDeadUnitsFromQueue()
{
    for (int i = _turnQueue.Count - 1; i >= 0; i--)
    {
        if (_turnQueue[i].IsDead) _turnQueue.RemoveAt(i);
    }
}
```

#### 주요 성과 및 해결 이유 (Why)

- **선택 이유 :** `TurnManager`의 `PrepareTurnQueue()` 단계에서 모든 생존 유닛에게 0~10000 사이의 `RandomSpeed`를 부여하여 동속도 유닛의 순서를 공정하게 랜덤화했습니다.
- **최종 선택 근거 :** 턴 완료 직후 `RemoveDeadUnitsFromQueue()`를 호출하고 `OnTurnQueueUpdated` 이벤트를 발생시킴으로써, 턴 도중 사망한 유닛이 즉각 UI와 큐에서 제외되도록 구조를 개선했습니다.
