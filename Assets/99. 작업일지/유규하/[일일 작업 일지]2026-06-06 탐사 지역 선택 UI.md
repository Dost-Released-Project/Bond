# 탐사 지역 선택 UI (2026-06-06)

탐사 준비(Embark) 오버레이 step1 지도 영역에 "탐사 지역 선택" 기능을 채워 넣음.
기존엔 지도 자리가 플레이스홀더("지도 — 추후 구현 예정")였고, ExpeditionPayload는 DungeonType을 하드코딩(`DungeonType.Ruin`)으로 받고 있었음.

## 한 일

### 1. 탐사 지역 데이터 모델
- 신규 `ExpeditionRegion` (순수 C#, `Bond.Expedition`): `Id / DisplayName / Meta / DungeonType`.
- `ExpeditionPayload`: `DungeonType DungeonType` 프로퍼티 → `ExpeditionRegion Region` 으로 교체.
  - **단, LeeJuno 맵 시스템(MapInitializer / MapGenerator / FixedMapGenerator)이 `payload.DungeonType` 을 직접 읽으므로 호환용 passthrough 유지:**
    `public DungeonType DungeonType => Region != null ? Region.DungeonType : DungeonType.None;`
  - 덕분에 맵 시스템 + HA 에디터 툴(ExpeditionPayloadWindow) 무수정.
  - `SetContents` 3번째 인자 `ExpeditionRegion` 으로 변경, `Clear()` 는 `Region = null`.

### 2. 지역 목록 공급
- 신규 인터페이스 `IExpeditionRegionProvider.GetRegions()`.
- 임시 구현 `TestExpeditionRegionProvider`: 하드코딩 2지역(속삭이는 숲/Forest, 무너진 성채/Ruin). **추후 DBSO 기반 구현으로 교체 예정**.
- `TownScope` 에 `Register<IExpeditionRegionProvider, TestExpeditionRegionProvider>(Scoped)` 등록, `EmbarkController` 생성자 주입.

### 3. UI
- `EmbarkController`: `SelectedRegion` 상태 + `OnRegionChanged` / `OnEmbarkBlocked` 이벤트 + `GetRegions` / `SelectRegion`.
  - `ConfirmEmbark`: 지역 미선택 시 `OnEmbarkBlocked` 발화 후 return(씬 로드 안 함).
  - `SavePayload`: `SelectedRegion` 전달(하드코딩 `Ruin` 제거).
- `EmbarkPresenter`: 지도 영역에 지역 카드(이름+메타) 빌드, 클릭→`SelectRegion`, 선택 카드 하이라이트(`--selected`), 목적지 정보 바 갱신. 미선택 출정 시 step2 footer 힌트를 적색(`--stress-crit`)으로 "탐사 지역을 먼저 선택하세요." 표시.
- UXML: `map-placeholder` → 제목 + ScrollView(`embark__region-list`).
- USS: `embark__region-card` / `--selected` / `:hover` 등. 테마 변수만 사용(하드코딩 색·폰트 없음).

## 결정 사항
- DungeonType passthrough로 LeeJuno 영역 무수정.
- 지역 데이터는 순수 C#(임시), 추후 DBSO 교체.
- 출정 차단은 출정 버튼(step2)에서만, 경고는 footer 힌트.
- `SelectedRegion` 은 오버레이를 닫아도 유지(마을 세션 내).

## 버그 수정 — 지역 카드 선택+호버 스타일
- **증상**: 지역 카드가 선택된 상태에서 호버하면 선택 테두리(`--border-active`)가 호버 테두리(`--border-highlight`)로 바뀜.
- **원인**: CSS 특이도. `.embark__region-card:hover`(클래스1 + 의사클래스1 = 0,2,0)가 `.embark__region-card--selected`(클래스1 = 0,1,0)보다 높아, 선택+호버 동시 상태에서 hover 규칙이 이김.
  - 로스터 카드는 embark 안에서 선택 스타일이 `.embark__step1 .roster-card--selected`(자손 결합자 = 0,2,0)로 작성돼 hover와 동급 → 소스 순서로 이겨서 유지됨(원래 의도된 게 아니라 특이도가 우연히 맞은 것).
- **수정**: `.embark__region-card--selected:hover` 셀렉터 추가(0,3,0)로 선택 스타일이 호버보다 우선하도록.
