# 해고 버튼 — 꾹 눌러 해고 + 시계방향 진행 테두리

2026-06-19 / 유규하

## 무엇을 만들었나

CharacterDetail UI 헤더(캐릭터 이름 왼쪽)에 **해고 버튼**을 추가했다.
3초간 꾹 누르면 `Roster.Fire()`로 해당 캐릭터를 해고한다. 누르는 동안 버튼 테두리가
12시 방향에서 **시계방향으로 채워지며** 진행도를 보여주고, 도중에 떼면 취소된다.
해고가 완료되면 Detail UI를 닫는다.

## 동작 흐름

- **홀드 판정** — `PointerDown`에서 포인터를 캡처하고(버튼 밖으로 커서가 벗어나도 `PointerUp`을
  받기 위함) 프레임 틱 스케줄러를 Resume. 진행도는 틱 횟수가 아니라 `Time.realtimeSinceStartup`로
  실제 경과 시간을 재서 3초를 판정. `PointerUp` / 캡처 해제 / 패널 닫힘 / 모드 변경 시 취소.
- **해고** — 3초 도달 → 대상 캐릭터를 지역 변수로 잡아두고 → Detail을 닫고(`Hide()` +
  `OnCloseRequested`, 기존 닫기 버튼과 같은 경로라 악세서리 가방까지 닫힘) → `_roster.Fire(victim)`.
  로스터 패널 카드는 기존 `OnCharacterRemoved` 구독으로 자동 제거된다.
- **진행 테두리** — 별도 오버레이 `VisualElement`(`fire-ring`, picking ignore)에
  `generateVisualContent` 콜백을 걸고 Painter2D로 사각형 둘레를 그린다. 매 틱 `MarkDirtyRepaint()`로
  다시 그려서 채워지는 애니메이션을 만든다.

버튼 자체는 `Button`이 아니라 **plain `VisualElement`** 로 만들었다. `Button`의 `Clickable`
매니퓰레이터가 포인터를 캡처해 우리 홀드 로직과 충돌할 수 있어서.

## 핵심 결정

### 1. 해고 노출 범위 — `FullEdit`만으로는 부족
Detail UI는 `FullEdit` 모드로 두 곳에서 열린다: **마을 관리**와 **탐사 준비(Embark)**.
그런데 Embark의 파티(`IPartyController`)는 `Roster.Fire`를 통보받지 않는다. 거기서 출진 예정
대원을 해고하면 로스터에선 지워지지만 출발 시 그대로 배치되는 **desync 버그**가 난다.
→ `Show()`에 `allowFire` 파라미터(기본 `false`)를 추가하고, **마을 관리(`TownUIController`)만
`true`** 로 넘긴다. 전투(`ReadOnly`)·탐사는 모드로 이미 제외.

### 2. 진행 테두리 — 원형 → 각진 사각형
처음엔 원형 버튼(`border-radius`) + Painter2D `Arc`로 했다(원호는 시작/끝 각도만으로 부분 그리기가
쉬움). 이후 사각형으로 바꾸면서 `Arc`(원만 그림) 대신 **사각형 둘레 경로**로 교체했다:
12시(위 변 중앙)에서 시계방향으로 꼭짓점(우상→우하→좌하→좌상→복귀)을 잇되, 각 변 길이를 누적하며
진행도에 해당하는 길이까지만 `LineTo`, 마지막 변은 중간에서 멈춘다.

### 3. 테두리 정렬 — `border-width` 한 값으로 제어
링 오버레이가 버튼의 테두리-박스를 정확히 덮도록 두 가지를 묶었다:
- 그리는 선 두께 = `_fireBtn.resolvedStyle.borderTopWidth` (코드에서 읽음)
- 링 인셋 = `-border-width`로 동기화 (`SyncFireRingInset`, `GeometryChangedEvent`에서 호출)

→ USS `border-width` **하나만** 바꾸면 선 두께와 정렬이 같이 따라온다. 진행 띠가 테두리 위에 정확히
겹친다(안쪽 침범·바깥 틈 없음).

### 4. 색 — 적색 테두리 3단계
- 평상시: `--border-danger` (`#4a1a1a`, 기존)
- 호버: `--border-danger-hover` (`#7a2e2e`, 신규)
- 그려지는 테두리: `--border-danger-strong` (`#b84a4a`, 신규)

밝기 ~40 → ~69 → ~107로 고르게 벌어지고, 셋 다 G=B인 정통 적색이라 주황빛 `--stress-crit`과 구분된다.
신규 변수 2개는 가이드의 2-파일 규칙대로 매니페스트 + 두 테마(DarkAmber/DarkBlue)에 같이 추가했다
(위험색은 테마 불문 적색이라 두 테마 동일 값).

배경·라벨도 적색 계열(`--slot-trait-bg` / `--slot-trait-dot`)이지만, 3단계로 나눈 건 "테두리"에 한정.

## 미해결 / 참고

- **에디터 미검증** — 컴파일과 USS 진단만 확인. 실제 플레이 테스트(마을 명부 우클릭 → 해고 홀드)는
  아직 안 함.
- **DarkBlue 테마 `--border-accent` 누락**(기존 버그, 이번 작업과 무관) — 매니페스트엔 선언됐지만
  DarkBlue엔 값이 없어 `transparent`로 폴백된다. 해고 버튼은 이번에 이 변수 의존을 떼서 영향은 없지만,
  DarkBlue를 실제로 쓸 거면 채워둘 것.
- **홀드 시간 `FIRE_HOLD_SECONDS = 3f`** 상수 — 조절은 이 값 하나로.
