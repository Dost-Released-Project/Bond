## 0. 출발점 — 성향 리액션 20종 평가

표의 성향(트레잇) 리액션 20개를 "트리거(조건) + 행동" 두 축으로 기존 리액션 시스템 대비 평가. MP·쿨다운은 구현 계획 없음(액션의 "무시" 문구는 no-op), 스트레스는 존재, 연쇄·봉인은 신규 구축 합의. 결과적으로 필요한 빌드 블록을 도출하고 ROI 순으로 구현.

## 1. 리액션 봉인 (Seal) + Composite

봉인 = **지속시간 있는 일시 비활성(상태이상)**. 리액션을 "소모"하는 게 아니라 일정 자기 턴 수 동안 잠갔다가 자동 해제. "설계 리액션 무시"의 지속 버전도 이걸로(DesignedOnly).

- `02. Scripts/ReactionSystem/ReactionSeal.cs` (신규) — `SealKind { All, DesignedOnly, Slots }` + `ReactionSeal`(Kind/Slots/RemainingTurns).
- `02. Scripts/ReactionSystem/BaseCharacter.Seals.cs` (신규 파셜) — `ApplySeal`/`IsReactionSealed`/`TickSeals`/`ActiveSeals`. `IsReactionSealed` 가 단일 기준(추후 우선순위-취소 확장 시 실행 직전 재확인에서도 재사용).
- `ReactionSystem.Resolve` — 봉인된 리액션은 발화 후보에서 제외(한 줄).
- `TurnManager` — 자기 턴 시작에 `TickSeals()`(버프 tick과 동일 자리).
- `ReactionEffect.cs` — `SealReactionEffect`(TargetSelf/Kind/Count/Duration) + `CompositeReactionEffect`(여러 효과 순차). Composite 가 "행동+봉인+스트레스" 묶음을 가능하게 함(리액션 1개당 Effect 1개 제약 해제).

설계 메모: 지속 `DurationTurns=1`=이번 라운드, `2`=다음 턴까지. 만료는 자기 턴 tick(버프와 동일). "N개 봉인"은 fire 시점에 비어있지 않은 슬롯 중 무작위 N개 확정.

**경계**: 봉인은 같은 이벤트 내 "이미 수집된" 다른 리액션을 즉시 취소하진 못함(지속 상태). 같은 이벤트 취소는 추후 (a) `ReactionExecution`에 Priority 추가 + (b) 실행 루프에서 `IsReactionSealed` 재확인으로 확장 예정.

## 2. 스트레스 · 포메이션 이동 효과

- `StressReactionEffect` (Delta>0 증가/<0 감소; Target=Self/Observed/…). 기존 API 부호 분기 — 증가=`ReduceInsanity`(명명과 반대), 감소=`RecoverInsanity`.
- `FormationMoveReactionEffect` (Front=0 / Back=3). 대상 슬롯 차 있으면 Swap.
- `02. Scripts/BattleSystem/BaseCharacter.Formation.cs` (신규 파셜) — `MoveToFormationIndex`, `GetOpposingByRank`(거리 적), `GetSameSideAllies`. 모두 자기 `m_formationManager` 사용.
- `E_TargetFilter` 에 `Self` 추가(효과 대상으로 리액터 자신).

## 3. 소형 조건/타겟 + 턴 스킵

- 조건: `HpAboveCondition`, `StressAboveCondition`(기본 50=경고), `PartyStressAverageCondition`(같은 진영 평균 > 임계, args 계약 변경 없이 `GetSameSideAllies` 사용).
- 거리 타겟: `E_TargetFilter.FrontmostEnemy/BackmostEnemy` + `SkillCastReactionEffect` 가 `reactor.GetOpposingByRank` 로 해석. (가까운=최전열 Rank1, 먼=최후열 Rank4)
- 턴 스킵: `02. Scripts/ReactionSystem/BaseCharacter.TurnState.cs`(신규) `RequestSkipTurns`/`ConsumeSkipTurn` + `SkipTurnReactionEffect` + `TurnManager` 가 자기 턴 시작에 소비(강제 휴식).

## 4. 자기턴 행동 오버라이드 (돌발행동 본체)

상태형 트리거(HP/스트레스 "~시")는 자기 턴에 평가되어 **계획 행동을 대체**한다.

- `E_ReactionPhase.OnSelfTurn`(신규) — 자기 턴에만 평가, 파이프라인 ReactionCall 에선 자동 제외(phase 불일치).
- `02. Scripts/ReactionSystem/BaseCharacter.Anomaly.cs`(신규) — `TryRunSelfTurnAnomalyAsync`/`FindSelfTurnAnomaly`. OnSelfTurn 트레잇 중 Subject 기반 조건 충족분의 효과 실행 → true(계획 행동 생략).
- `TurnManager` — tick → 강제휴식 → **돌발 판정** 순. 발동 시 `TakeTurnAsync` 생략(플레이어 캐릭터도 입력 전에 가로채짐 = 의도된 "설계 스킬 무시").

제약: OnSelfTurn 조건은 Subject 기반(HpBelow/HpAbove/StressAbove/PartyStressAverage)만 사용. 자기 턴엔 원본 스킬 컨텍스트가 없어 SkillType/Crit/Evade 등은 무의미(평가 시 try/catch 가드).

## 5. 연쇄 카운트

- `02. Scripts/ReactionSystem/BaseCharacter.Chain.cs`(신규) — `ReactionsFiredCount`/`IncrementReactionCount`/`ResetReactionCount`. "연속" = 마지막 자기 턴 이후 발동 수(관찰용 카운터, 리액션을 제한/차단하지 않음).
- `ReactionCall`(BattlePipeLineSo) — 리액션 발동마다 +1.
- `TurnManager` — 자기 턴 도달 시(돌발 판정 후) 리셋.
- `ReactionCountCondition`(≥N) + `ResetChainCountReactionEffect`(같은 진영 카운트 리셋, 006용).
- 013(쉽게 지침)은 자가 보정: 탈진 발동 직후 자기 턴 리셋이 돌아 매 턴 연속 발동 안 함.

## 6. 돌발 확률 공식 (관계 보너스 제외)

- `BaseCharacter.GetAnomalyChance()` = `clamp([기본 0.2 + Insanity×0.005] − Stat.INT×0.004, 최저 0.05, 1)`. **관계항 제외**(정책). 상수는 밸런스 튜닝 대상.
- `RollAnomaly()`, `IsTraitReaction()`.
- `ReactionSystem.Resolve` — **성향(트레잇) 리액션은 돌발 굴림 성공 시에만 발동(Anomaly)**, **역할(설계) 리액션은 항상 발동(Success)**. → "성향은 확률적, 설계는 의도대로".
- `IsSuccess` 스텁 + `using Random` 별칭 제거.
- OnSelfTurn 돌발도 동일 굴림으로 게이트 → 100% 발동 문제 해결.
- BondAwakening(유대적 각성)은 관계/유대 의존이라 자연히 보류(굴림은 Anomaly/미발동 이분).

## 7. 아군 돌발 관찰 (016 트리거)

"아군 돌발 발생 시"를 자기턴 근사로 구현(즉발 방송 이벤트는 telegraph와 같은 부류라 제외).

- `BaseCharacter` 최근 돌발 플래그 — `HasRecentAnomaly`/`MarkAnomaly`/`ClearRecentAnomaly`(Anomaly 파셜). 돌발 시 set, 자기 턴 시작에 clear.
- 마킹: `ReactionCall`(Anomaly 결과) + `TryRunSelfTurnAnomalyAsync`.
- `AllyAnomalyCondition` — 같은 진영 아군(자기 제외)이 최근 돌발했는지.

## 빌더 어휘 (누적 DSL)

`using static Reactions.Authoring.ReactionDefBuilder;`

- **조건**: `SubjectIs` · `SkillTypeIs` · `HpBelow` · `HpAbove` · `StressAbove` · `PartyStressAbove` · `ReactionCountAtLeast` · `AllyAnomaly` · `Crit` · `Evaded` · `Hit` · `Killed` · `DamageAtLeast` · `DamageAtLeastMaxHp`
- **행동**: `CastSkill` · `Intercept` · `Buff` · `Seal` · `Composite`(또는 `.Do(a,b,c)`) · `Stress` · `MoveBack`/`MoveFront` · `SkipTurn` · `ResetChainCount`
- **편집 슬롯**: `ObserveTarget` · `ActionSkill`
- **타겟**(E_TargetFilter): None · Caster · Target · Observed · Self · FrontmostEnemy · BackmostEnemy
- **페이즈**: None · PreApply · PostApply · OnSelfTurn
- **SealKind**: All · DesignedOnly · Slots

대표 예:
```csharp
// 008 복수심: 본인 피격 → 공격자 반격 + 역할 리액션 봉인
Def("TRT_008", RoleType.None).Phase(E_ReactionPhase.PostApply).Observe(E_ObserveFilter.Self)
  .When(SubjectIs(E_TargetFilter.Target), Hit())
  .Do(CastSkill(E_TargetFilter.Caster), Seal(SealKind.DesignedOnly, turns: 1));

// 001 겁쟁이: 자기 턴 HP30%↓ → 후열 이탈 + 전체 봉인
Def("TRT_001", RoleType.None).Phase(E_ReactionPhase.OnSelfTurn).Observe(E_ObserveFilter.Self)
  .When(HpBelow(0.3f))
  .Do(MoveBack(), Seal(SealKind.All, turns: 1));

// 016 규율적: 아군 돌발 후 내 턴 → 자기통제(리액션 1개 봉인)
Def("TRT_016", RoleType.None).Phase(E_ReactionPhase.OnSelfTurn).Observe(E_ObserveFilter.Self)
  .When(AllyAnomaly())
  .Do(Seal(SealKind.Slots, turns: 1, count: 1));
```
