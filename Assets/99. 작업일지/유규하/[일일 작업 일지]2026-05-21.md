## 3. 주요 변경 사항

### CharacterDetailViewMode (신규)
```csharp
public enum CharacterDetailViewMode
{
    FullEdit,   // 마을 — 역할·리액션·장비 모두 편집 가능
    EquipOnly,  // 탐사 중 (전투 외) — 장비 교체만 가능
    ReadOnly,   // 전투 중 — 열람 전용
}
```

### CharacterDetailController (신규)
- `SetCharacter / SetRole / EquipAccessory / UnequipAccessory`
- `SetReactionTrigger / SetReactionSkill / SetReactionTarget`
- `GetAvailableSkills / GetAllProfessionSkills / GetRoleTriggers / GetPartyMembers`
- 이벤트: `OnCharacterSet / OnRoleChanged / OnAccessoryChanged / OnReactionChanged`
- 생성자: `(CharacterItemService, DataBaseSO skillDb, Roster)`

### EquipSlots 컴포넌트 (신규)
- **EquipSlots.uxml** — 72×72 칩 4개 (무기·방어구·부속품×2)
- **EquipSlots.uss** — 칩 스타일, 장착 강조선, 드롭 타겟 하이라이트, disabled 처리
- **EquipSlotsPresenter** — 드래그·드롭 연동 (`InventoryTransferService`), 툴팁, 우클릭 해제 이벤트

### CharacterDetail.uxml (수정)
- body를 ScrollView + 좌우 2단 레이아웃으로 채움
- 좌측 컬럼: 초상화 / 정체성(클래스·레벨·광기·역할버튼) / 능력치(STR·AGI·INT) / 전투(HP·DEF·ATK·SPD) / 성향 목록 / 장비 슬롯
- 우측 컬럼: 스킬 그리드 / 리액션 슬롯 6개 (역할 0~1 + 성향 2~5)
- `<Style src="../Shared/EquipSlots.uss" />` 로 EquipSlots 스타일 로드
- `<ui:Template src="../Shared/EquipSlots.uxml" />` + `<ui:Instance>` 로 EquipSlots 마운트

### CharacterDetailPresenter (전면 재작성)
- `Show(BaseCharacter, CharacterDetailViewMode, IInventory)` — 열 때 모드·인벤토리 명시
- `SetViewMode(CharacterDetailViewMode)` — 런타임 전환 (탐사 진입 시 EquipOnly 등)
- 역할 버튼 클릭 → `controller.SetRole()`
- 리액션 슬롯 스킬 레이블 클릭 → 스킬 피커 드롭다운 표시 후 `controller.SetReactionSkill()`
- EquipSlotsPresenter 위임: 장착은 AccessoryBagView 드래그로, 해제는 우클릭으로

### TownScope (수정)
```csharp
var skillDb = Addressables.LoadAssetAsync<DataBaseSO>("SkillDataBase").WaitForCompletion();
builder.Register<CharacterDetailController>(Lifetime.Scoped)
    .WithParameter("skillDb", skillDb);
```

## 4. 에디터 설정 항목

### 4-1. Addressables — SkillDataBase 키 확인 (필수)

TownScope에서 `"SkillDataBase"` 키로 `DataBaseSO`를 로드합니다.

**확인 방법:**
1. Unity 메뉴 → `Window > Asset Management > Addressables > Groups`
2. `DataBaseSO` 에셋(스킬 데이터가 담긴 것) 선택
3. Inspector에서 **Addressable** 체크박스 활성화 확인
4. Address 필드가 정확히 `SkillDataBase` 인지 확인

> 키가 다르면 `WaitForCompletion()` 반환값이 null이 되어 CharacterDetailController 등록 시 오류 발생.

---

### 4-2. AccessoryBagView — 씬 배치 확인 (필수)

TownScope가 `RegisterComponentInHierarchy<AccessoryBagView>()`로 주입합니다.  
**마을 씬 Hierarchy에 AccessoryBagView 컴포넌트가 부착된 GameObject가 있어야** 합니다.

> 이미 배치되어 있다면 추가 작업 없음.

---

### 4-3. TownUIController — Inspector 연결 확인 (필수)

| Inspector 필드 | 연결 대상 |
|----------------|-----------|
| `Town UI Document` | 마을 씬의 UIDocument 컴포넌트 (TownUI.uxml 사용 중인 것) |

> 기존에 이미 연결되어 있다면 추가 작업 없음.

---

### 4-4. CharacterDetail.uxml — 경로 해상도 확인 (권장)

Unity Editor에서 **CharacterDetail.uxml**을 UI Builder로 열었을 때:
- `EquipSlots.uss` 스타일시트가 정상 로드되는지 확인 (좌측 StyleSheets 패널)
- EquipSlots 칩 4개가 장비 섹션에 표시되는지 확인

경로 문제가 있다면 Style 태그 경로를 절대 경로로 변경:
```xml
<Style src="project://database/Assets/UI/Shared/EquipSlots.uss" />
<ui:Template name="EquipSlots" src="project://database/Assets/UI/Shared/EquipSlots.uxml" />
```

---

### 4-5. EquipSlots.uss — Panel Settings 확인 (선택)

프로젝트 Panel Settings에서 글로벌 스타일시트로 `EquipSlots.uss`를 등록하면 경로 참조 없이도 동작합니다.  
`CharacterDetail.uxml`의 `<Style>` 태그로 이미 처리했으므로 **보통 불필요**합니다.
