# [일일 작업 일지]2026-05-19

## 1. 작업 개요
- **전투 파이프라인 리팩토링:** `BattlePipeLineSo`의 `EntryStep`에서 스킬 타입에 따라 스탯을 분기 처리하고, 전체 파이프라인이 개별 타겟(Per-Target) 단위로 실행되도록 구조를 개선.
- **일지 시스템 - 탐사 인벤토리 연동:** `JournalSystem`에서 아이템 획득 선택지("Take")를 골랐을 때 탐사 인벤토리(`ExpeditionInventory`)에 아이템이 실제 추가되도록 메타데이터 및 외부 핸들러 연동.

---

## 2. 문제·변경·성과

### 2-1. 전투 파이프라인 개별 타겟(Per-Target) 구조 개편
- **Before:**
  - `EntryStep`에서 마법/힐 등 스킬 종류와 무관하게 시전자의 `atk` 스탯만 참조함.
  - `BattleManager`가 다수의 `targets`를 포함한 단일 `BattleContext`를 넘겨 파이프라인이 1회만 동작. 이로 인해 광역기 사용 시 타겟별로 회피/치명타 등 개별 판정을 세밀하게 적용하기 불가능했음.
- **After:**
  - `EntryStep`에 `switch`문을 도입하여 `SkillType` (OFFENSIVE/SPELL/SUPPORT/DEFENSIVE)에 맞춰 각각 `atk`, `Sp_Atk`, `INT`, `def`를 참조하도록 개선.
  - `BattleContext`가 단일 `target`만을 가지도록 변경. `BattleManager`의 `ApplyAct`에서 `GetTargets()`로 유효 타겟들을 산출한 후, 타겟의 수만큼 파이프라인 `Run()`을 반복 호출하도록 로직 개편.
  - `EvasionStep`, `CriticalStep`, `DefenseStep`, `ApplyStep`에서 개별 타겟의 `AGI`, `def` 스탯을 참조하도록 고도화. (단, SUPPORT 스킬은 회피 및 방어력 차감 대상에서 제외)
- **Why:** 
  - 각 스킬의 논리적 정확도 향상.
  - 광역기 피격 시 어떤 적은 회피하고, 어떤 적은 크리티컬을 입는 등 각 객체별 독립적인 전투 판정을 수행하여 전투의 사실감과 확장성(DLV - Logic Blindness)을 확보하기 위함.

### 2-2. 일지 선택지와 탐사 인벤토리 연동
- **Before:** 
  - `JournalSystem`이 선택지 액션을 실행할 때 `actionKey` 문자열 하나만 전달되어, 핸들러 측에서 어떤 아이템을 지급해야 할지 컨텍스트(ID 등)를 알 수 없었음.
- **After:** 
  - `JournalReport`에 `Dictionary<string, string> Metadata` 필드를 추가하여 순수 데이터 내에 ItemId 등 컨텍스트를 보존.
  - `IJournalActionHandler` 인터페이스의 시그니처를 `ExecuteAction(string actionKey, JournalReport report)`로 확장하여 메타데이터를 참조할 수 있게 함.
  - `JournalInventoryActionHandler`를 신규 구현하여 "Take" 액션 발생 시 `ExpeditionPayload.Supplies.AddItemId()`를 호출하도록 연결.
  - VContainer `RootScope_KWT`에 신규 핸들러 주입.
- **Why:** 
  - `JournalSystem` 본연의 데이터(Text/Icon) 처리 역할과 외부 시스템(Inventory) 제어 역할을 분리하면서도(DLV 아키텍처), 단절된 데이터 흐름을 메타데이터로 이어주어 확장성 확보.

## 3. 상세 사용 가이드

### 3-1. 전투 파이프라인
* **스탯 적용:** 앞으로 힐 스킬을 만들 때는 `SkillType`을 `SUPPORT`로 설정하면 자동으로 시전자의 `INT` 수치를 기반으로 회복량이 계산됩니다.
* **로그 확인:** 인게임 전투 테스트 시 콘솔 창에 `[EntryStep]`, `[EvasionStep]`, `[DefenseStep]` 등의 prefix로 각 개별 타겟(예: "오크1", "오크2")별로 독립적인 회피율 및 데미지 차감 로그가 출력됩니다.

### 3-2. 일지 이벤트 등록 (아이템 지급)
* **메타데이터 세팅:** LocationEvent 또는 임의의 일지 데이터 생성 시 아이템을 지급하고 싶다면, `JournalReport.Metadata["ItemId"]`에 지급할 아이템 ID (예: `"ITEM_MAP_01"`)를 넣어주면 됩니다.
* **테스트 방법:** `JournalTestRunner.cs`가 켜진 씬에서 키보드 **[H]** 키를 누르면 "낡은 지도(ITEM_MAP_01)" 발견 이벤트가 발생합니다. 이후 인게임 UI에서 "챙긴다(Take)"를 선택하면 탐사용 가방(`ExpeditionPayload.Supplies`)에 아이템이 즉시 추가됩니다.

## 4. 문제 재현 및 검증 방법
1. **전투 파이프라인 검증:**
   - 1대다 광역 스킬을 사용하는 캐릭터 배치.
   - 적 대상의 AGI, def 수치를 각각 다르게 세팅.
   - 스킬 시전 시 에디터 콘솔(Console)에 출력되는 회피(Evaded) 및 크리티컬(Critical) 여부가 대상마다 다르게 뜨는지, 데미지가 방어력에 따라 다르게 깎이는지 확인.
2. **일지 연동 검증:**
   - 에디터 플레이 모드 시작.
   - 키보드 `H` 키 입력으로 이벤트 트리거.
   - 일지 UI에서 "챙긴다" 클릭.
   - `ExpeditionPayloadWindow` 창 등에서 `Supplies` 슬롯에 해당 아이템(예: ITEM_MAP_01) 1개가 정상적으로 카운팅되었는지 확인.
