# [일일 작업 일지] 2026-06-17

## 1. 작업 개요
* 스킬 시트 재조정을 통해 Value 및 Effect 정의
* 데미지 판정 텍스트 연출 추가 
* 회피(Evade) 판정 발생 시 화면에 "빗나감" 텍스트 연출 적용.
* "빗나감" 텍스트 색상을 Unity Inspector에서 자유롭게 제어할 수 있도록 설정 필드 노출.

---

## 2. 문제 · 변경 · 성과 (Before-After & Why)

### 2-1. Before
* 공격이 회피되더라도 UI 상에 아무런 시각적 피드백이 발생하지 않았음.
* 빗나감 연출을 단순 텍스트로 지정할 시, 코드 내부에 텍스트 내용("빗나감")과 색상(`Color.white`)이 고정(하드코딩)되어 기획적 수정이 불가한 상태였음.

### 2-2. After
1. **이벤트 체인 구축**: `BaseCharacter`에 `OnEvaded` 이벤트 신설, 파이프라인 `EvasionStep`에서 회피 판정 시 이를 트리거하여 `CharacterSlot`이 감지하도록 연결.
2. **비주얼 연출 추가**: `TextVisualizer`에 `ShowMiss()`를 설계하여 "빗나감" 텍스트를 연출하도록 구현.
3. **색상 외제화**: `TextVisualizer`에 `[SerializeField] private Color m_MissColor` 필드를 연동하여 인스펙터에서 빗나감 텍스트의 색상을 유연하게 조절할 수 있도록 개선.

### 2-3. Why (타당성)
* 유저에게 전투 상황(회피인지, 방어력에 의한 0 데미지인지)을 즉각적으로 구분할 수 있는 피드백이 반드시 필요함.
* 게임 테마나 폰트 렌더링 방식의 변경에 유연하게 대응하기 위해 색상값을 직렬화 필드로 제어하는 설계 방식을 유지함.

---

## 3. 상세 사용 가이드
1. 씬(Scene) 내 캐릭터 슬롯 프리팹 혹은 인스턴스의 `CharacterSlot` 오브젝트 하위에 배치된 `TextVisualizer` 컴포넌트 확인.
2. `TextVisualizer` 컴포넌트의 **Miss Color** 속성을 원하는 색상(기본값: 흰색)으로 변경하여 사용 가능.

---

## 4. 테스트 및 검증 방법
1. 캐릭터의 회피 스탯을 임시 조정하거나 `BattlePipeLineSo.cs` 내 `EvasionStep`의 명중 확률 판단 부근에서 `context.isEvaded = true;` 강제 입력.
2. 전투를 개시하여 공격이 회피되었을 때 "빗나감" 문구와 지정한 색상으로 연출(상승 및 페이드아웃)이 올바르게 실행되는지 관찰.
