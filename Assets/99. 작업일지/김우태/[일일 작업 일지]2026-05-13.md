# [일일 작업 일지]2026-05-13
**작업자:** AI Agent

## 1. 작업 개요
- **목표:** 기 구현된 TSV 임포터 툴을 활용하여 저널 시스템의 실사용 데이터 임포트 파이프라인 연동.
- **관련 시스템:** DataSheet TSV Importer, JournalSystem, VContainer (RootScope)

## 2. 문제·변경·성과
### Before
- `JournalParser.cs`가 제네릭 `DataBaseSO`를 강제로 생성 및 할당하고 있었음.
- `LocationEventProvider`에서 하드코딩된 단일 `JournalDataSO` 참조를 들고 있었음.
- VContainer(`RootScope.cs`)에서 일지 시스템 의존성이 주석 처리되어 있었음.

### After
- **Why (이유):** 다른 파이프라인(Class, Skill 등)과 통일성을 맞추어 `JournalDataBaseSO`를 별도로 생성. 동적 데이터 로드가 가능하도록 Provider 로직을 변경함.
- `JournalDataBaseSO.cs` 생성 (전용 DB).
- `JournalParser.cs`가 `JournalDataBaseSO` 타입으로 Asset을 만들고 등록하도록 변경.
- `LocationEventProvider`가 생성자에서 `JournalDataBaseSO`를 주입(Inject)받고, `SetDiscovery` 발생 시 동적으로 `_journalDB.GetSO<JournalDataSO>(ID)`를 호출하여 템플릿을 완성하도록 개선.
- `RootScope.cs`에 Addressables를 활용한 동기 로드 및 `LocationEventProvider` 주입 코드 활성화.

## 3. 상세 사용 가이드
1. **스프레드시트 세팅:** `Journal` 시트를 생성하고 `ID`, `Paragraphs`, `Options`, `IconPath` 헤더를 입력.
2. **구글 시트 연동:** `Assets/Data/GoogleSheetConfig` 폴더에 `ConfigSample(journal).asset` (타입: GoogleSheetConfig) 생성 후 시트 ID 입력.
3. **임포트 & 어드레서블 자동화:** 툴에서 임포트를 진행하면 `Assets/Data/GeneratedSO/JournalDataBase.asset`이 자동 생성되며, `AddressableHelper`를 통해 어드레서블에 자동으로 등록됩니다. (어드레스 키: `JournalDataBase`)
4. **실행:** 게임 시작 시 `RootScope`가 자동으로 DB를 로드하며, 이벤트 발생 시 `LocationEventProvider.SetDiscovery(이벤트ID, 장소명, 아이템)` 형태로 호출.

## 4. 문제 재현 방법 (테스트)
- TSV 임포트 시 `ID` 값이 누락되었을 경우, `LocationEventProvider`가 `Warning` 로그(`[LocationEventProvider] 일지 데이터를 찾을 수 없습니다. ID: ...`)를 발생시킴.
- VContainer에서 `JournalDataBase` Addressables 키 누락 시, `[RootScope] JournalDataBase 를 로드할 수 없습니다.` 에러 발생.
