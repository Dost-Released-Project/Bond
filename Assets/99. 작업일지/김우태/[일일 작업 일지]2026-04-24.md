- **주요 작업 영역 :** BattleSystem / Slot
- **작업 목표 :** 로직과 비주얼의 결합도를 낮추고, 확장 가능한 슬롯 시스템 구축

    - `BaseBinder`, `INeedBind`를 이용한 범용 바인딩 시스템 구축
    - `CharacterSlot`(Logic)과 `CharacterSlotVisualizer`(Visual) 구현
    - `SlotBinder`를 통한 자동 컴포넌트 연결 로직 적용

## 2. 문제 사항 / 변경 사항 / 주요 성과

### 📌 이슈 #1 : 로직과 비주얼의 강한 결합 문제 해결

#### 문제 사항
- **문제 설명 :** 기존 방식에서는 로직 클래스가 특정 비주얼 클래스를 직접 참조하여, 비주얼이 변경될 때마다 로직 코드를 수정해야 하는 번거로움이 있었음.

#### Before / After

| 구분 | 내용 |
|------|------|
| **Before** | `CharacterSlot` 내에서 `CharacterSlotVisualizer`를 직접 참조하여 메서드 호출 |
| **After** | `ICharacterSlotVisualizer` 인터페이스를 통해 통신하고, `SlotBinder`가 두 객체를 연결함 |

```csharp
// Before
public class CharacterSlot : MonoBehaviour {
    public CharacterSlotVisualizer visualizer; // 직접 참조
    void Update() { visualizer.SetColor(color); }
}

// After
public class CharacterSlot : MonoBehaviour, INeedBind<ICharacterSlotVisualizer> {
    private ICharacterSlotVisualizer m_visualizer; // 인터페이스 참조
    public void Bind(ICharacterSlotVisualizer visual) => m_visualizer = visual;
}
```

#### 주요 성과 및 해결 이유 (Why)

- **선택 이유 :** 인터페이스를 활용하여 로직은 "무엇을 그릴지"만 결정하고, 비주얼은 "어떻게 그릴지"만 담당하게 하여 유지보수성을 높임.
- **고려했던 다른 방법 :** 싱글톤 또는 전역 이벤트 시스템을 통한 통신.
- **최종 선택 근거 :** 슬롯은 다수가 생성되는 객체이므로, 개별 객체 단위로 명확한 연결을 보장하는 Binder 패턴이 성능과 관리 측면에서 가장 유리함.

## 3. 상세 시스템 사용 가이드

### 🛠️ Slot 시스템 사용 가이드

#### 개요
- **대상 :** 전투 시스템 내 캐릭터 슬롯 프리팹
- **목적 :** 슬롯의 상태 변화에 따른 시각적 피드백 제공 및 데이터 관리

#### 사용 방법

**Step 1. 컴포넌트 구성**  
슬롯 역할을 할 GameObject에 다음 컴포넌트를 모두 추가합니다.
- `CharacterSlot` (Logic)
- `CharacterSlotVisualizer` (Visual)
- `SlotBinder` (Connection)

**Step 2. 데이터 설정**  
`CharacterSlot`의 `Color Data` 필드에 해당 슬롯의 상태별 색상 정보(Normal, Hover, Pressed)를 할당합니다.

**Step 3. 자동 연결 확인**  
`SlotBinder`가 `Awake` 시점에 동일 오브젝트 내의 로직과 비주얼을 찾아 서로 연결(Bind)해줍니다.

#### 주의 사항

- ⚠️ **컴포넌트 누락 주의** : `SlotBinder`는 동일 오브젝트 내에 로직과 비주얼이 모두 있어야 정상 작동합니다. 누락 시 Warning 로그가 출력됩니다.
