# [일일 작업 일지] 2026-05-24

## 1. 작업 개요
- **목표:** 스킬 시스템의 다중 효과(Multi-Effect) 지원 리팩토링 및 타겟팅 오류("대상 없음") 디버깅, 사망 판정 시스템 뼈대 구현.
- **주요 내용:** 
  - `ApplyStep`의 조건문을 제거하고 전략 패턴(Strategy Pattern)을 활용한 `CompositeSkill`과 이펙트 객체(`HpChangeEffect`, `BuffEffect`) 도입.
  - VContainer 의존성 주입 에러 해결 (`RootScope` 글로벌 등록으로 중앙화).
  - 스킬 데이터(TSV) 비트마스크 해석의 진영별 시각적 대칭성 확보를 위한 파서 롤백 및 교정.
  - 사망 처리 시 발생하는 `NullReferenceException` 예외 방어 및 파이프라인 가시성(Debug Log) 강화.

---

## 2. 문제·변경·성과 (Before & After)

### A. 스킬 파이프라인 다형성 리팩토링
- **Before:** `ApplyStep` 내부에 거대한 `switch`문이 존재하여 효과가 추가될 때마다 파이프라인 코드를 수정해야 했음. (개방-폐쇄 원칙 위배)
- **After:** `CompositeSkill`이 시트의 `Effects` 컬럼을 읽어 필요한 효과 객체(`HpChangeEffect` 등)를 리스트로 담고, 순차적으로 실행하도록 변경.
- **Why:** 기획자가 TSV 시트 데이터만으로 새로운 조합의 스킬을 자유롭게 만들 수 있도록 Data-Driven 아키텍처를 실현하기 위함.

### B. VContainer 의존성 에러 (ISkillManager 미등록)
- **Before:** `TurnManager`나 `BattleStageEntry`가 `ISkillManager`를 주입받지 못해 `VContainerException` 발생. (`SkillScope`가 분리되어 있어 스코프 트리를 찾지 못함)
- **After:** 불필요해진 `SkillScope`를 삭제하고, `RootScope`에 `SkillDataBaseSO`를 연결하여 `SkillManager`를 전역 서비스(Singleton)로 등록.
- **Why:** 스킬 데이터는 프로젝트 전반에서 참조되어야 하는 순수 데이터이므로, 특정 씬의 스코프에 묶이는 것보다 Root 단에서 글로벌하게 관리하는 것이 구조적으로 안정적이기 때문.

### C. 몬스터 스킬 "대상 없음" 오류 및 파서 교정
- **Before:** 기획자는 TSV에 `1100`을 적을 때 화면과 동일하게 "왼쪽부터 전열(Rank 1, 2)"을 의도했으나, 파서는 "오른쪽 끝이 Rank 1"인 2진수 표준을 따라 후열(Rank 3, 4)로 잘못 번역. (이 격차를 메꾸려 억지로 반전 로직(`MirrorMask`)을 넣었다가 꼬임 발생)
- **After:** 억지 반전 로직을 모두 롤백(정상화)하고, 파서(`SkillDataParser`)를 진영별로 분리.
  - 플레이어 진영(`UseableSlots`, `AllyTargetMask`): 오른쪽부터 읽음 (`PlayerSideMaskConverter`)
  - 적군 진영(`EnemyTargetMask`): 왼쪽부터 읽음 (`EnemySideMaskConverter`)
- **Why:** 기획자의 시각적 직관(게임 화면의 좌우 대칭)을 컴퓨터의 데이터 처리와 완벽하게 일치시키기 위함.

### D. 사망 시 NullReferenceException 방어 및 로그 강화
- **Before:** 스킬 공격으로 대상이 사망 시 진영 슬롯(`CurrentSlot`)이 `null`로 초기화되는데, 이후 스킬 이펙트 강조를 해제하려다 널 참조 에러가 발생하여 게임 흐름이 정지됨.
- **After:** 강조 해제 로직에 Safe Navigation 연산자(`?.`) 추가 (`target.CurrentSlot?.SetForceClick(false);`). `EvasionStep`, `DefenseStep`, `ApplyStep`에 진행 상황을 알 수 있는 명확한 컬러 디버그 로그 추가.
- **Why:** 예외로 인한 전투 흐름 정지를 막고, 파이프라인 진행 상태(회피, 방어 무효화 등)에 대한 가시성을 확보하여 추후 디버깅을 용이하게 하기 위함.

## 3. 상세 사용 가이드

**[스킬 시트 작성 가이드 (Effects 컬럼)]**
*   **새로운 스킬 효과 조합:** `Skill.tsv`의 `Effects` 컬럼에 번호 또는 문자열을 콤마(`,`)로 구분하여 입력.
    *   예시: `1, 2` (또는 `HpChange, Buff`) 입력 시, HP 증감과 버프가 동시에 들어가는 복합 스킬 생성.

**[비트마스크(Rank) 작성 가이드]**
*   **시각적 대칭 작성법:** 게임 화면(`[4][3][2][1] VS [1][2][3][4]`)에 보이는 그대로 기입.
    *   `UseableSlots` / `AllyTargetMask` (아군 타겟): 화면 왼쪽 기준이므로 `0011` 입력 시 전열(1, 2열) 의미.
    *   `EnemyTargetMask` (적군 타겟): 화면 오른쪽 기준이므로 `1100` 입력 시 전열(1, 2열) 의미.

## 4. 문제 재현 및 검증 방법
1. 유니티 에디터에서 `Skill.tsv`를 수정하고 Reimport하여 `SkillDataBaseSO`가 갱신되었는지 확인.
2. `Assets/05. RootVContainer/RootVContainer.prefab` 인스펙터에 `SkillDataBaseSO`가 연결되어 있는지 확인.
3. `Assets/03. PipeLine/BattlePipeLineSO.asset` 인스펙터의 Steps 리스트 맨 하단에 `ApplyStep`이 추가되어 있는지 확인.
4. 전투 진입 후 몬스터(GoblinAssassin)가 정상적으로 타겟을 잡아 스킬을 시전하고, "대상 없음" 또는 "NullReferenceException"이 발생하지 않는지 콘솔 로그 확인.
