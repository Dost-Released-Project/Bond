# [일일 작업 일지] 2026-05-15

## 1. 작업 개요
- **작업명**: 일지(Journal) 시스템 UI 개선 및 다중 페이지/지연 실행 아키텍처 구현
- **작업 목표**: 
    1. 첫 페이지 이전 버튼의 시각적 비활성화 처리.
    2. 단일 제공자(Provider)가 여러 장의 일지를 생성할 수 있는 구조로 개선.
    3. 선택지 결과를 즉시 실행하지 않고 마지막 장에서 일괄 실행(지연 실행) 및 외부 시스템 연동 구조 구축.

## 2. 문제 · 변경 · 성과 (Before - After + Why)

### 2.1. 버튼 비활성화 시각 처리 및 동기화
- **문제 (Before)**: `SetEnabled(false)` 호출 시 기능은 막히나 시각적으로 활성화된 것처럼 보였으며, 인게임에서 `ObservableValue`의 초기값 중복으로 인해 UI 갱신이 발생하지 않음.
- **변경 (After)**: 
    - `Journal.uss`에 `:disabled` 가상 클래스와 `background-image: none` 속성 추가.
    - `JournalBinder.Start()`에서 현재 모델 상태를 UI에 강제 동기화하는 로직 추가.
- **이유 (Why)**: 사용자에게 명확한 상호작용 피드백을 제공하고, 초기 진입 시 데이터-UI 간 불일치 문제를 해결하기 위함.

### 2.2. 다중 리포트(페이지) 지원 구조
- **문제 (Before)**: `LocationEventProvider`가 단일 변수 구조여서 여러 번 이벤트를 발생시켜도 마지막 하나만 표시됨.
- **변경 (After)**: 
    - `IJournalContentProvider`의 반환형을 `IEnumerable<JournalReport>`로 변경.
    - `LocationEventProvider` 내부에 리스트 버퍼를 도입하여 모든 이벤트를 독립된 페이지로 생성.
    - `JournalSystem`에서 `SelectMany`를 통해 모든 페이지를 수집하도록 수정.
- **이유 (Why)**: 하루 동안 발생한 다양한 사건들을 누락 없이 각각의 일지 페이지로 제공하기 위함.

### 2.3. 선택지 결과 지연 실행 및 외부 연동 (DIP 적용)
- **문제 (Before)**: 선택지를 누르는 즉시 결과가 반영되어 '이전 장'으로 돌아가 선택을 바꿔도 결과가 중복/오류 발생함. 또한 `JournalSystem`이 구체적인 외부 로직에 의존함.
- **변경 (After)**: 
    - `JournalReport`에 `SelectedOption` 임시 저장 공간 마련.
    - `IJournalActionHandler` 인터페이스 도입 및 `JournalSystem`에서 이를 통한 제어 위임.
    - 마지막 페이지 확인 시에만 모든 선택 결과를 일괄 실행하도록 변경.
- **이유 (Why)**: 플레이어에게 선택 번복 기회를 제공하고, 일지 시스템이 인벤토리나 스탯 등 외부 시스템을 직접 알지 못하게 하여 결합도를 낮추기 위함(DIP).

## 3. 문제 재현 방법
1. 첫 페이지에서 이전 버튼이 활성화된 것처럼 보이는 현상 확인 (USS 수정 전).
2. `JournalTestRunner`에서 'H' 키를 연달아 눌러도 일지가 한 페이지만 뜨는 현상 확인 (버퍼 도입 전).

## 4. 상세 사용 및 확인 가이드 (테스트 케이스 가이드)
- **테스트 1**: 'H' 키 입력 시 일지가 최소 2장 이상 생성되는지 확인.
- **테스트 2**: 첫 장에서 이전 버튼이 회색 비활성 상태인지 확인.
- **테스트 3**: 각 페이지에서 선택지를 고르고 이전 장으로 돌아가 선택을 바꾼 뒤, 마지막 장에서 '닫기' 클릭 시 콘솔에 최종 선택 결과만 일괄 출력(`MockJournalActionHandler` 로그)되는지 확인.
