# **기획 의도 기준**

본 기획서의 모든 시스템은 다음 세 가지 기획 의도에 부합하도록 설계되어야 합니다. 각 시스템을 구현하거나 검토할 때 아래 기준을 참고해 주세요.

| \# | 기획 의도 | 시스템적 구현 방향 |
| :---- | :---- | :---- |
| 1 | 플레이어 서사 중심에서 벗어나 다양한 캐릭터들 사이의 서사를 제공합니다. | 리더는 관리자일 뿐이며, 대원 간의 관계 변화가 게임플레이에 직접 영향을 미쳐야 합니다. |
| 2 | 캐릭터의 성격을 텍스트가 아닌 실제 리액션 로직과 행동으로 데이터화하여 제공합니다. | 성향(Trait)은 설명 텍스트가 아닌, 전투에서 실제로 발동하는 행동 데이터로 구현됩니다. |
| 3 | 플레이어가 초기에는 지적 쾌감을, 후기에는 스스로 쌓아올린 서사를 느끼게 합니다. | 리액션 로직 설계의 재미와, 누적된 파티 관계 데이터가 서사로 전환되는 구조를 만들어야 합니다. |

**✦  구현 체크리스트 — 캐릭터 직업 및 역할/성향**

| ☐ | **INT 통제력 계수** | 캐릭터 INT 수치 | Anomaly\_Rate 억제값 \+ 각성 전환 확률 | 1 |

| ☐ | **역할군 추가 트리거 활성화** | 리더가 부여한 역할군 타입 | 활성화된 추가 Trigger\_ID 리스트 | 2 |
| ☐ | **Role\_Table 구성** | 역할군 타입 | 역할군별 Trigger\_ID 리스트 데이터 | 2 |

| ☐ | **Resolve\_Reaction() 모듈** | 이벤트 발생 \+ 전체 캐릭터 슬롯 스캔 | 조건 충족 슬롯의 판정 실행 신호 | 4 |

# **캐릭터 직업 및 역할/성향**

*기획 의도: 캐릭터 시스템은 기획 의도 2번(성격의 데이터화)의 직접적인 구현입니다. 성향(Trait)은 전투 리액션 로직과 결합할 때 비로소 행동으로 증명됩니다.*

## **1\. 직업 및 스탯 성장**

| 직업 | 성장 특화 및 특징 |
| :---- | :---- |
| 전사 (Warrior) | STR 성장 특화. 물리 공격력과 생존력 중심 성장 |
| 도적 (Rogue) | AGI 성장 특화. 행동 순서와 치명타율 중심 성장 |
| 성직자 (Cleric) | INT 성장 특화. 리액션 통제력(지능 판정)과 지원 위력 중심 성장 |

* **지능(INT) 통제력:** 지능 수치가 높을수록 스트레스에 의한 Anomaly\_Rate(돌발 행동 확률)를 억제하며, 판정 실패 시 Bond\_Awakening(유대적 각성)으로 전환될 확률을 보정합니다.

## **2\. 역할군 시스템**

리더(유저)가 파티를 구성할 때 각 캐릭터에게 부여하는 전술적 임무입니다. 이 설정에 따라 추가적인 리액션 조건이 개방됩니다.

| 역할군 | 추가 활성화 트리거 |
| :---- | :---- |
| 탱커 (Tanker) | \[피격 시\], \[아군 위기 시\] |
| 딜러 (Dealer) | \[적 처치 시\], \[치명타 발생 시\] |
| 서포터 (Supporter) | \[아군 턴 종료 시\], \[상태이상 발생 시\] |

## **3\. 성향 시스템**

성향은 캐릭터가 태생적으로 보유한 고유의 '사건 감시 조건'입니다. 설명 텍스트로 존재하는 것이 아니라, 특정 상황에서 실제로 발동하는 행동 데이터입니다.

| 구성 요소 | 설명 |
| :---- | :---- |
| 내장 트리거 | 성향 자체에 고정된 감시 조건 (예: '체력이 30% 이하일 때', '적군이 증원될 때') |
| 스킬 결합 슬롯 | 유저가 획득한 스킬을 해당 성향 트리거에 1:1로 매칭 |
| 결합 원리 | \[역할군 트리거 \+ 스킬\]과 \[성향 트리거 \+ 스킬\]을 조합하여 한 캐릭터 내 다중 리액션 구조를 설계 |

## **4\. 지능 판정과 돌발 행동**

트리거(역할군 or 성향)가 만족되는 순간 시스템은 다음 로직을 실행합니다.

| 단계 | 내용 |
| :---- | :---- |
| 1\. Event Detect | 설정된 트리거 조건 충족 |
| 2\. Probability Calculation | Fail\_Chance \= (Trait\_Instability \+ Stress) \- (INT\_Factor \+ Role\_Stability) |
| 3-A. Success (이성) | 유저가 커스텀 설정한 스킬 정상 실행 |
| 3-B. Fail (본능) | 해당 성향에 내장된 Anomaly\_Skill 강제 실행 |
| 4\. 유대적 각성 (조건부) | 판정 실패 시 유대 관계가 높은 아군이 있으면 돌발 행동을 취소하고 강화 스킬 실행 |

## **5\. 데이터 테이블 구성**

| 테이블명 | 내용 |
| :---- | :---- |
| Class\_Table | 직업별 레벨당 스탯 가산치 데이터 |
| Role\_Table | 역할군 부여 시 활성화될 Trigger\_ID 리스트 |
| Trait\_Table | 성향 고유 트리거 및 판정 실패 시 발동할 Anomaly\_Skill\_ID |

### **캐릭터 객체 구현 가이드**

* **Character Class:** List\<ReactionSlot\>을 관리하며, 각 슬롯은 \[출처: 역할군 or 성향\], \[트리거: Trigger\_ID\], \[실행: User\_Skill\_ID\] 데이터를 포함합니다.

* **Battle Engine:** 이벤트 발생 시 모든 캐릭터의 슬롯을 스캔하여 조건이 맞을 경우 Resolve\_Reaction() 모듈로 판정을 진행합니다.
