# **기획 의도 기준**

본 기획서의 모든 시스템은 다음 세 가지 기획 의도에 부합하도록 설계되어야 합니다. 각 시스템을 구현하거나 검토할 때 아래 기준을 참고해 주세요.

| \# | 기획 의도 | 시스템적 구현 방향 |
| :---- | :---- | :---- |
| 1 | 플레이어 서사 중심에서 벗어나 다양한 캐릭터들 사이의 서사를 제공합니다. | 리더는 관리자일 뿐이며, 대원 간의 관계 변화가 게임플레이에 직접 영향을 미쳐야 합니다. |
| 2 | 캐릭터의 성격을 텍스트가 아닌 실제 리액션 로직과 행동으로 데이터화하여 제공합니다. | 성향(Trait)은 설명 텍스트가 아닌, 전투에서 실제로 발동하는 행동 데이터로 구현됩니다. |
| 3 | 플레이어가 초기에는 지적 쾌감을, 후기에는 스스로 쌓아올린 서사를 느끼게 합니다. | 리액션 로직 설계의 재미와, 누적된 파티 관계 데이터가 서사로 전환되는 구조를 만들어야 합니다. |

**✦  구현 체크리스트 — 캠핑 시스템**

### **리더의 일지**

| 완료 | 구현 항목 | Input | Output | 참고 |
| :---: | ----- | ----- | ----- | ----- |
| ☐ | **일지 텍스트 생성 시스템** | 전투 중 발생한 이벤트 데이터 | 텍스트 풀에서 조합된 리더의 일지 문장 | 1 |
| ☐ | **전투 보고 / 관계 기록 / 보급 현황 탭** | 전투 결과 데이터 \+ 관계 수치 변화 | 탭별 일지 텍스트 출력 | 1 |

### **보급 및 전술 세팅**

| 완료 | 구현 항목 | Input | Output | 참고 |
| :---: | ----- | ----- | ----- | ----- |
| ☐ | **식량 배급 3단계 선택 로직** | 유저의 배급 방식 선택 (전원/절약/굶기) | 대원별 스트레스 증감량 \+ HP 변화량 | 2 |
| ☐ | **치료 우선순위 및 INT 보너스** | 유저의 치료 대상 선택 \+ 약품 수량 | HP 회복량 \+ 다음 전투 INT 보너스 적용 | 2 |
| ☐ | **성향-역할군-스킬 재설정 UI** | 유저의 슬롯 변경 입력 | 업데이트된 List\<ReactionSlot\> | 3 |
| ☐ | **Battle\_Start\_Modifier 클래스** | 캠프 배급/치료 결과값 | 다음 전투 시스템 초기값 (스트레스/HP/INT 보정치) | 4 |

# **캠핑 시스템**

*기획 의도: 기획 의도 1번(캐릭터 간 서사)과 3번(서사의 축적)이 실현되는 공간입니다. 전투에서 발생한 사건들이 리더의 일지를 통해 서사로 전환되고, 플레이어는 이를 바탕으로 다음 전투를 설계합니다.*

## **1\. 리더의 일지**

캠프 진입 시 가장 먼저 나타나는 인터페이스로, 지난 전투의 결과와 현재 파티의 물리적/정신적 상태를 텍스트 형식으로 출력합니다.

### **전투 보고 요약**

*보급품 변화: "지난 행군과 전투로 식량 3유닛 소모. 현재 잔여 보급품으로 2일간 버티기 가능."*

*상태 보고: "용병 A의 부상이 깊음. 스트레스 수치가 임계점에 도달하여 다음 전투 시 돌발행동 가능성 높음"*

### **관계 관찰 기록**

캐릭터 간의 직접 대화 대신, 리더가 관찰한 파티 내 기류를 일지 형식으로 기록합니다.

*긍정적 변화: "전투 중 용병 A가 B를 구출한 사건 이후, 두 사람 사이에 묘한 신뢰가 싹틈. (관계 수치 상승)"*

*부정적 변화: "용병 C의 잦은 실수에 대해 D가 노골적인 불만을 표시함."*

## **2\. 보급 및 정비 로직**

| 선택지 | 효과 |
| :---- | :---- |
| 전원 배급 | 스트레스 크게 감소 |
| 절약 배급 | 스트레스 현상 유지 |
| 굶기 | 스트레스 대폭 상승 및 HP 감소 |

* **치료 우선순위:** 한정된 약품을 어떤 대원에게 사용할지 결정합니다. 치료받은 대원은 다음 전투에서 지능(INT) 판정 보너스를 얻습니다.

## **3\. 파티 전술 세팅**

캠프의 최종 단계로, 일지에서 파악한 파티원들의 상태를 고려하여 \[성향-역할군-스킬\] 조합을 재설정합니다.

* **상태 고려 예시:** "A와 B의 관계가 나빠졌으므로, A의 성향 트리거에 B의 협동 스킬을 배치하는 것은 리스크가 큼."

* **역할군 재배정:** 부상당한 딜러를 탱커 역할군으로 변경하여 리액션 트리거 조건을 \[공격\]에서 \[피격\] 위주로 변경 가능합니다.

## **4\. 기술 구현 명세**

### **일지 텍스트 생성 시스템**

* **데이터 소스:** 전투 중 발생한 Reaction\_Success\_Count, Take\_Damage\_Event, Stress\_Level 등의 변수를 취합합니다.

* **텍스트 조립:** 미리 정의된 텍스트 풀(Pool)에서 조건에 맞는 문장을 조합하여 출력합니다.

*예: if (Bond\_A\_B \> 50\) print("{Name\_A}와 {Name\_B}의 호흡이 눈에 띄게 좋아졌다.");*

### **캠프 효과 데이터 전이**

캠프에서 결정된 배급 상태와 치료 결과는 Battle\_Start\_Modifier 클래스에 저장되어 다음 전투 시스템의 초기값으로 로드됩니다.

## **5\. UI 연출 가이드**

* **시각 연출:** 오래된 양피지 혹은 군용 수첩 느낌의 텍스트 UI. 리더의 필기체 폰트와 간소한 스케치 삽입

* **인터랙션:** 일지의 페이지를 넘길 때마다 \[자원 요약\] → \[관계 기록\] → \[전술 세팅\] 단계로 자연스럽게 이동

*기획 의도: 일지는 개발자가 쓴 이야기가 아닙니다. 전투 중 실제로 발생한 데이터가 텍스트로 조립되어 출력됩니다. 플레이어는 자신이 플레이한 결과가 서사로 돌아오는 경험을 합니다(의도 3번).*
