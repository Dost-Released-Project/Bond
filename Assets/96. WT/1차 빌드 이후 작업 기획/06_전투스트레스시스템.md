# [코드 작업 기획서] 전투 이벤트 조건에 따른 스트레스 수치 변동 결과의 유저 예측성 설계 및 검증

## 1. 개요 및 주제
* **주제**: 전투 중 특정 조건(피격, 사망, 치명타, 회피, 리액션) 발생 시 캐릭터 스트레스 변동 결과를 유저가 예측 가능하게 설계하고 검증할 수 있는가?
* **목표**: 전투 이벤트별 스트레스(`Insanity`) 변동 트리거 구현 및 0~100 범위 경계값 제한 동작 보증

## 2. 세부 동작 예측 설계 (규칙 및 시각 피드백)

### 2.1. 스트레스 변동 트리거 규칙

| 트리거 ID | 이벤트 조건 | 대상 | 변동 수치 | 시각 피드백 연출 (UI) |
|---|---|---|---|---|
| **STR_TRG_HIT** | 일반 피격 | 피격자 | +3 | 머리 위 붉은색 `+3 ⚡` 출력 |
| **STR_TRG_CRIT_HIT** | 치명타 피격 | 피격자 | +10 | 머리 위 크고 굵은 붉은색 `+10 ⚡` 및 화면 흔들림 |
| **STR_TRG_ALLY_DEAD** | 아군 사망 | 생존 아군 전체 | +20 | 생존 아원 전체 머리 위 붉은색 `+20 ⚡` 순차 출력 |
| **STR_TRG_MISS** | 공격 회피당함 | 시전자 | +5 | 머리 위 보라색 `+5 ⚡` 출력 |
| **STR_TRG_CRIT_ATK** | 치명타 공격 성공 | 시전자 | -5 | 머리 위 녹색 `-5 💖` 출력 |
| **STR_TRG_POSITIVE_REACTION**| 긍정 리액션 발동 | 아군 전체 | -5 | 아군 전원 머리 위 녹색 `-5 💖` 출력 |

* **수치 범위 보증 (Clamp)**: 모든 누적 연산 후 `Mathf.Clamp(value, 0, 100)` 처리
* **UI 연동**: 수치 변동 시 `BaseCharacter.OnInsanityChanged` 이벤트를 발행하여 UI 게이지 즉시 동기화

## 3. 동작 검증 시나리오 (Test Scenarios)

### 시나리오 A: 일반/치명타 피격 시 수치 누적 검증
* **목표**: 피격 타입에 따라 설계된 수치가 정확히 누적 및 연출되는가?
* **테스트**:
  1. 캐릭터 A(스트레스 0)가 일반 피격 받음 -> 스트레스 3 변동 확인, `+3 ⚡` UI 연출 확인
  2. 캐릭터 A가 치명타 피격 받음 -> 스트레스 13 변동 확인, `+10 ⚡` UI 연출 확인

### 시나리오 B: 아군 사망 시 생존자 스트레스 동시 상승 검증
* **목표**: 아군 사망 시점에 생존자 전원에게 누적 처리가 전파되는가?
* **테스트**:
  1. 생존 캐릭터 B, C 존재
  2. 아군 캐릭터 D 사망 -> B, C의 스트레스 수치 각각 +20 확인, 연출 확인

### 시나리오 C: 공격 실패 및 성공 판정 피드백 검증
* **목표**: 회피 및 치명타 성공에 따른 스트레스 가감 산출이 정확한가?
* **테스트**:
  1. 캐릭터 E가 공격 시도, 적이 회피함 -> E의 스트레스 +5 확인
  2. 캐릭터 E가 치명타 공격 성공함 -> E의 스트레스 -5 확인

### 시나리오 D: 긍정 리액션의 아군 전체 영향 검증
* **목표**: 긍정 리액션 발동 시 아군 전원의 스트레스가 경감되는가?
* **테스트**:
  1. 캐릭터 F가 긍정 리액션 발동 -> 아군 전체 스트레스 -5 확인

### 시나리오 E: 경계값(0 이하, 100 이상) 처리 예외 검증
* **목표**: 수치 변동 결과가 0~100 범위를 벗어나지 않는가?
* **테스트**:
  1. 스트레스 95 상태에서 치명타 피격(+10) 시 최종 100 확인 (105 초과 차단)
  2. 스트레스 3 상태에서 치명타 성공(-5) 시 최종 0 확인 (-2 미만 차단)
